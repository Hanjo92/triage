Part A. Railway 실제 배포 따라하기
1) 배포 전에 준비할 것
Telegram BotFather에서 발급받은 BOT_TOKEN
GitHub 저장소 1개
Python 프로젝트 루트에 bot.py, requirements.txt, Procfile 또는 start command
PostgreSQL 연결 문자열을 받을 준비
2) 최소 폴더 구조
bot.py
requirements.txt
.env.example
README.md
3) requirements.txt 예시
python-telegram-bot==21.10psycopg2-binary==2.9.9python-dotenv==1.0.1
4) Railway 환경변수
BOT_TOKEN=<텔레그램 토큰>DATABASE_URL=<Railway PostgreSQL URL>APP_ENV=production
5) 시작 커맨드
python bot.py
6) Railway 배포 순서
단계
해야 할 일
체크포인트
1
Railway에서 New Project → Deploy from GitHub Repo 선택
저장소 연결 성공
2
PostgreSQL 서비스 추가
DATABASE_URL 발급
3
Variables에 BOT_TOKEN, DATABASE_URL 입력
민감정보가 코드에 하드코딩되지 않음
4
Start Command를 python bot.py로 설정
Deploy 로그에 import 오류 없음
5
텔레그램에서 /start 테스트
응답 메시지 수신
6
checkin / completion 저장 확인
DB row 생성
7) 첫 배포 후 바로 확인할 것
봇이 /start 에 응답하는지
버튼 클릭 후 다음 단계로 정상 이동하는지
DB에 users, checkins, completions row가 쌓이는지
에러 발생 시 Railway 로그에 traceback이 남는지
8) 운영 팁
초기에는 polling으로 시작하고, 안정화 후 webhook 전환
추천 액션은 코드 상수보다 DB 테이블에서 읽도록 유지
알림 기능은 배포 안정화 후 2차로 붙이는 편이 안전
실험 단계에서는 retention보다 복귀율 데이터를 먼저 본다

적용 규칙
회복 모드에서는 '해야 한다'보다 '가볍게, 다시, 괜찮다' 계열 표현을 우선 사용
압박 상태에서는 결과 지향 문구보다 '첫 클릭, 첫 줄, 첫 3분'처럼 진입 행동을 강조
부분 완료 피드백은 사용자의 죄책감을 낮추고 연결감을 높이는 방향으로 유지
같은 액션도 상황별 문구를 2~3개 로테이션하면 반복 피로가 줄어든다
